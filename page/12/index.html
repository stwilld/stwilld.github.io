1034. 边框着色

发表于 2021-04-25 | 分类于 leetcode

难度 medium

链接：https://leetcode-cn.com/problems/coloring-a-border

题目描述：

给出一个二维整数网格 grid，网格中的每个值表示该位置处的网格块的颜色。

只有当两个网格块的颜色相同，而且在四个方向中任意一个方向上相邻时，它们属于同一连通分量。

连通分量的边界是指连通分量中的所有与不在分量中的正方形相邻（四个方向上）的所有正方形，或者在网格的边界上（第一行/列或最后一行/列）的所有正方形。

给出位于 (r0, c0) 的网格块和颜色 color，使用指定颜色 color 为所给网格块的连通分量的边界进行着色，并返回最终的网格 grid 。

示例 1：

输入：grid = [[1,1],[1,2]], r0 = 0, c0 = 0, color = 3
输出：[[3, 3], [3, 2]]
示例 2：

输入：grid = [[1,2,2],[2,3,2]], r0 = 0, c0 = 1, color = 3
输出：[[1, 3, 3], [2, 3, 3]]
示例 3：

输入：grid = [[1,1,1],[1,1,1],[1,1,1]], r0 = 1, c0 = 1, color = 2
输出：[[2, 2, 2], [2, 1, 2], [2, 2, 2]]

提示：

1 <= grid.length <= 50
1 <= grid[0].length <= 50
1 <= grid[i][j] <= 1000
0 <= r0 < grid.length
0 <= c0 < grid[0].length
1 <= color <= 1000

解题思路：

这道题目描述得太晦涩了，其实意思就是找连通区域的边界进行着色。连通区域的边界就是grid网格的边界或者是连通区域与其他区域相连的位置。如下图所示：

黄色的为连通分量，褐黄色的为边界的两种情况，一种直接是grid的边界，一种是和其他颜色的矩形块相连。我们要把连通分量的所有边界进行染色，更换成指定的颜色color；这道题目的解法还是比较直接的，暴力解法，用深度优先搜索，对于一个位置grid[row][col]，我们用一个visited矩阵标记它被访问过， 我们遍历它的四个方向，如果某个方向超出grid边界，那当前位置肯定是grid边界，我们用flag进行标记（flag初值为false，此处置为true）。如果是访问过的，直接continue，否则说明我们遍历的这个方向的位置在grid内部且未曾访问过，如果它的颜色不等于连通分量的统一颜色，我们同样用flag进行标记，表示当前位置是边界，否则我们遍历到的位置是连通分量的一部分，可能是边界也可能是内部，我们在这个新位置上调用dfs，进行遍历。随后，检查flag，如果为true，说明当前遍历的位置是边界，保存到一个ArrayList里面，当dfs结束后，我们将ArrayList里面的边界值取出来，对grid进行染色并且返回即可。

代码： java t82 s5

1446-连续字符

发表于 2021-04-24 | 分类于 leetcode

难度 easy

链接：https://leetcode-cn.com/problems/consecutive-characters

题目描述：

给你一个字符串 s ，字符串的「能量」定义为：只包含一种字符的最长非空子字符串的长度。

请你返回字符串的能量。

示例 1：

输入：s = “leetcode”
输出：2
解释：子字符串 “ee” 长度为 2 ，只包含字符 ‘e’ 。
示例 2：

输入：s = “abbcccddddeeeeedcba”
输出：5
解释：子字符串 “eeeee” 长度为 5 ，只包含字符 ‘e’ 。
示例 3：

输入：s = “triplepillooooow”
输出：5
示例 4：

输入：s = “hooraaaaaaaaaaay”
输出：11
示例 5：

输入：s = “tourist”
输出：1

提示：

1 <= s.length <= 500
s 只包含小写英文字母。

解题思路：

这道题目没有什么难度，直接暴力解法可以求得，用res记录最终结果，也就是最长连续字符的长度，用一个变量tmp记录当前遍历到的字符的最长长度，当s.charAt(i)==s.charAt(i-1)的时候，tmp加一，否则tmp重置为一，然后用tmp动态更新res，使其为res和tmp中的较大值即可。

代码： java t100 s43

1561. 你能获得的最大硬币数

发表于 2021-04-24 | 分类于 leetcode

难度 medium

链接：https://leetcode-cn.com/problems/maximum-number-of-coins-you-can-get

题目描述：

有 3n 堆数目不一的硬币，你和你的朋友们打算按以下方式分硬币：

每一轮中，你将会选出 任意 3 堆硬币（不一定连续）。
Alice 将会取走硬币数量最多的那一堆。
你将会取走硬币数量第二多的那一堆。
Bob 将会取走最后一堆。
重复这个过程，直到没有更多硬币。
给你一个整数数组 piles ，其中 piles[i] 是第 i 堆中硬币的数目。

返回你可以获得的最大硬币数目。

示例 1：

输入：piles = [2,4,1,2,7,8]
输出：9
解释：选出 (2, 7, 8) ，Alice 取走 8 枚硬币的那堆，你取走 7 枚硬币的那堆，Bob 取走最后一堆。
选出 (1, 2, 4) , Alice 取走 4 枚硬币的那堆，你取走 2 枚硬币的那堆，Bob 取走最后一堆。
你可以获得的最大硬币数目：7 + 2 = 9.
考虑另外一种情况，如果选出的是 (1, 2, 8) 和 (2, 4, 7) ，你就只能得到 2 + 4 = 6 枚硬币，这不是最优解。
示例 2：

输入：piles = [2,4,5]
输出：4
示例 3：

输入：piles = [9,8,7,6,5,1,2,3,4]
输出：18

提示：

3 <= piles.length <= 10^5
piles.length % 3 == 0
1 <= piles[i] <= 10^4

解题思路：

这道题目，我用了最简单的思路去思考的，首先对数组进行排序，记长度为len，其中最小的前1/3那一部分，肯定是Bob取走的，不用考虑，而剩下的有序的2/3部分，就是Alice和你进行分配的，每次你要取走一堆硬币，肯定要搭配上Alice取走更多硬币，所以，能搭配的方式只有，Alice取走第len-1堆的硬币，你取走第len-2堆的硬币， Alice取走第len-3堆的硬币，你取走len-4堆的硬币，……，所以从len/3这个位置开始，你就开始取，然后跳过Alice该取的那一堆，直到把数组遍历完为止。其实按这个思路来的话，应该是一道easy题。

代码： java t31 s37

1448-统计二叉树中好节点的数目

发表于 2021-04-24 | 分类于 leetcode

难度 medium

链接：https://leetcode-cn.com/problems/count-good-nodes-in-binary-tree

题目描述：
给你一棵根为 root 的二叉树，请你返回二叉树中好节点的数目。

「好节点」X 定义为：从根到该节点 X 所经过的节点中，没有任何节点的值大于 X 的值。

 

示例 1：

输入：root = [3,1,4,3,null,1,5]
输出：4
解释：图中蓝色节点为好节点。
根节点 (3) 永远是个好节点。
节点 4 -> (3,4) 是路径中的最大值。
节点 5 -> (3,4,5) 是路径中的最大值。
节点 3 -> (3,1,3) 是路径中的最大值。
示例 2：

输入：root = [3,3,null,4,2]
输出：3
解释：节点 2 -> (3, 3, 2) 不是好节点，因为 “3” 比它大。
示例 3：

输入：root = [1]
输出：1
解释：根节点是好节点。

提示：

二叉树中节点数目范围是 [1, 10^5] 。
每个节点权值的范围是 [-10^4, 10^4] 。

解题思路：
这道题目思路比较直接，用深度优先遍历的方法，用一个val表示到从根节点到当前节点的遍历过的节点的最大值，初始化为Integer.MIN_VALUE，然后进入递归函数，如果node.val>=val，说明当前节点是好节点，则将全局变量global加一，然后如果左子节点存在，对其调用递归函数，其中val用val和当前node的val中的较大值进行更新，如果右子节点存在，也是进行同样的操作。当所有节点遍历完，返回count即可。

代码： java t100 s41

1502-判断能否形成等差数列

发表于 2021-04-24 | 分类于 leetcode

难度 easy

链接：https://leetcode-cn.com/problems/can-make-arithmetic-progression-from-sequence

题目描述：
给你一个数字数组 arr 。

如果一个数列中，任意相邻两项的差总等于同一个常数，那么这个数列就称为 等差数列 。

如果可以重新排列数组形成等差数列，请返回 true ；否则，返回 false 。

 

示例 1：

输入：arr = [3,5,1]
输出：true
解释：对数组重新排序得到 [1,3,5] 或者 [5,3,1] ，任意相邻两项的差分别为 2 或 -2 ，可以形成等差数列。
示例 2：

输入：arr = [1,2,4]
输出：false
解释：无法通过重新排序得到等差数列。

提示：

2 <= arr.length <= 1000
-10^6 <= arr[i] <= 10^6

解题思路：
这道题目没有难度，先排个序，然后判断后一个数和前一个数的差是否一直不变即可。

代码： java t98 s68
