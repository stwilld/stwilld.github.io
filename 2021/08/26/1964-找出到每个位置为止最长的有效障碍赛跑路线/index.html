1964. 找出到每个位置为止最长的有效障碍赛跑路线

发表于 2021-08-26 | 分类于 leetcode

难度 hard
你打算构建一些障碍赛跑路线。给你一个 下标从 0 开始 的整数数组 obstacles ，数组长度为 n ，其中 obstacles[i] 表示第 i 个障碍的高度。

对于每个介于 0 和 n - 1 之间（包含 0 和 n - 1）的下标 i ，在满足下述条件的前提下，请你找出 obstacles 能构成的最长障碍路线的长度：

你可以选择下标介于 0 到 i 之间（包含 0 和 i）的任意个障碍。
在这条路线中，必须包含第 i 个障碍。
你必须按障碍在 obstacles 中的 出现顺序 布置这些障碍。
除第一个障碍外，路线中每个障碍的高度都必须和前一个障碍 相同 或者 更高 。
返回长度为 n 的答案数组 ans ，其中 ans[i] 是上面所述的下标 i 对应的最长障碍赛跑路线的长度。

示例 1：

输入：obstacles = [1,2,3,2]
输出：[1,2,3,3]
解释：每个位置的最长有效障碍路线是：

  • i = 0: [1], [1] 长度为 1
  • i = 1: [1,2], [1,2] 长度为 2
  • i = 2: [1,2,3], [1,2,3] 长度为 3
  • i = 3: [1,2,3,2], [1,2,2] 长度为 3
    示例 2：

输入：obstacles = [2,2,1]
输出：[1,2,1]
解释：每个位置的最长有效障碍路线是：

  • i = 0: [2], [2] 长度为 1
  • i = 1: [2,2], [2,2] 长度为 2
  • i = 2: [2,2,1], [1] 长度为 1
    示例 3：

输入：obstacles = [3,1,5,6,4,2]
输出：[1,1,2,3,2,2]
解释：每个位置的最长有效障碍路线是：

  • i = 0: [3], [3] 长度为 1
  • i = 1: [3,1], [1] 长度为 1
  • i = 2: [3,1,5], [3,5] 长度为 2, [1,5] 也是有效的障碍赛跑路线
  • i = 3: [3,1,5,6], [3,5,6] 长度为 3, [1,5,6] 也是有效的障碍赛跑路线
  • i = 4: [3,1,5,6,4], [3,4] 长度为 2, [1,4] 也是有效的障碍赛跑路线
  • i = 5: [3,1,5,6,4,2], [1,2] 长度为 2

提示：

n == obstacles.length
1 <= n <= 105
1 <= obstacles[i] <= 107

解题思路：
这道题目其实还是很简单的，先进行问题转换，对于输入数组obstacles，我们需要返回一个等长度的数组res，而res[i]的含义是以obstacles[i]为末尾的最长非递减子序列，这里可以参考NC91最长递增子序列，这是一道两天前才做过的题目，可以好好参考一下，解决这个问题用的是动态规划+二分查找的思路，这里我们也可以用这种思路，只不过在NC91最长递增子序列中二分查找我们找的是大于等于某个数target的第一个数，然后用target去替代找到的这个数，这种情况下是保证严格递增的，而在这道题目中，由题意可知，除第一个障碍外，路线中每个障碍的高度都必须和前一个障碍相同或者更高，因此我们要找的是严格大于某个数target的第一个数，然后用target去替代这个数，这样的话就保证不会把重复的元素去掉。这一步只需要在二分查找的过程中在判断的时候加上一个等号即可。我解这一道题目的时候一开始思路都是对的，但是犯了一个错误，就是每遍历一个obstacles[i]的时候，我都从0开始建立最长非递减子序列，因此浪费了很多时间，其实我们维护的最小非递减子序列seq是通用的，每进来一个新的元素，我们按照规则将其添加到seq中或者替换掉其中某个元素，进行维护即可。
