1372. 二叉树中的最长交错路径

发表于 2021-04-13 | 分类于 leetcode

难度 medium
给你一棵以 root 为根的二叉树，二叉树中的交错路径定义如下：

选择二叉树中 任意 节点和一个方向（左或者右）。
如果前进方向为右，那么移动到当前节点的的右子节点，否则移动到它的左子节点。
改变前进方向：左变右或者右变左。
重复第二步和第三步，直到你在树中无法继续移动。
交错路径的长度定义为：访问过的节点数目 - 1（单个节点的路径长度为 0 ）。

请你返回给定树中最长 交错路径 的长度。

示例 1：

输入：root = [1,null,1,1,1,null,null,1,1,null,1,null,null,null,1,null,1]
输出：3
解释：蓝色节点为树中最长交错路径（右 -> 左 -> 右）。
示例 2：

输入：root = [1,1,1,null,1,null,null,1,1,null,1]
输出：4
解释：蓝色节点为树中最长交错路径（左 -> 右 -> 左 -> 右）。
示例 3：

输入：root = [1]
输出：0

提示：

每棵树最多有 50000 个节点。
每个节点的值在 [1, 100] 之间。

解题思路：这道题其实不难的，但我就是以为那个node.val应该用起来，所以一直没有写出来，其实应该用一个形参len来记录长度，然后一直用len一直更新res即可，我们用dfs的思路解决问题，用一个变量dir来记录方向，len来记录走到当前节点的路径长度，然后dfs函数内，根据dir的不同分类讨论，注意当dir=0的时候，我们应该对左边节点调用dfs函数，并且len参数应该加1，此时是一条连续的路径，而此时，在这种情况下，如果右节点不为0，还需要对右节点调用dfs，且len为1，此时相当于把当前节点作为起始节点的情况考虑进来，因为在深度搜索的时候，我们并不知道哪个节点开始的交错路径长度是最长的，所以要随时把新纳入的节点作为起始节点进行考虑，这样才能遍历完所有情况。

代码 t55 s54 java
