1793. 好子数组的最大分数

发表于 2021-04-22 | 分类于 leetcode

难度 hard
给你一个整数数组 nums （下标从 0 开始）和一个整数 k 。

一个子数组 (i, j) 的 分数 定义为 min(nums[i], nums[i+1], …, nums[j]) * (j - i + 1) 。一个 好 子数组的两个端点下标需要满足 i <= k <= j 。

请你返回 好 子数组的最大可能 分数 。

示例 1：

输入：nums = [1,4,3,7,4,5], k = 3
输出：15
解释：最优子数组的左右端点下标是 (1, 5) ，分数为 min(4,3,7,4,5) * (5-1+1) = 3 * 5 = 15 。
示例 2：

输入：nums = [5,5,4,5,4,1,1,1], k = 0
输出：20
解释：最优子数组的左右端点下标是 (0, 4) ，分数为 min(5,5,4,5,4) * (4-0+1) = 4 * 5 = 20 。

提示：

1 <= nums.length <= 105
1 <= nums[i] <= 2 * 104
0 <= k < nums.length

解题思路：这道题是用贪心+双指针的方法完成的，看了题解。主要思想是，找到以nums[k]为最小值的好子数组的最大分数，并且不断更新nums[k]的值。用两个指针分别指向k的左边和右边的子数组中大于等于k的数，如果到达边界或者小于k，则指针不再移动。此时先更新一下好子数组的最大分数，然后进行判断，如果left=0&& right =len，那说明二者都是遇到边界停止了，那就break掉，如果left遇到边界，那nums[k]更新为nums[right]，如果right=len，那nums[k]更新为nums[left]，如果二者都没有遇到边界，此时说明以nums[k]为中心的好子数组已经得到最大值了，需要选择nums[left]和nums[right]中的较大值更新nums[k]，为什么是较大值呢？主要是使用较大值的话，可以包含更多情况，也就是此时在另一边默认是停止的，只需要更新较大值这一边即可，而如果选择较小值作为新的阈值的话，以较大值为阈值这种情况就被排除在外了，很可能会错过最优选择。
