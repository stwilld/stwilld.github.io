剑指offer59. 滑动窗口的最大值

发表于 2021-04-23 | 分类于 剑指offer

给定一个数组和滑动窗口的大小，找出所有滑动窗口里数值的最大值。

例如，如果输入数组 {2, 3, 4, 2, 6, 2, 5, 1} 及滑动窗口的大小 3，那么一共存在 6 个滑动窗口，他们的最大值分别为 {4, 4, 6, 6, 6, 5}。

解题思路：这里用了最直接的思路解题的，就是用一个函数helper计算i到j之间的最大值，然后开始遍历num数组，当然不是每个都计算一次，这样太费时了，基本上就是维护一个max_idx，如果当前遍历到的i和max_idx的距离在size以内，而且当前元素比max_idx对应的小，那就把max_idx对应的数加到res中，如果当前元素比max_idx对应的大，那就更新max_idx为当前i，并将num[max_idx]添加到res中；如果i和max_idx的差值已经不在size范围内了呢？那没办法，只能重新调用helper函数计算一次max_idx了，这样一直遍历下去直到数组遍历完。
代码 t90 s15 java

解题思路2：再看一下csnote上面提供的一个解法，用优先队列（大顶堆）来解，堆顶始终是最大的元素，当遍历到一个新元素的时候，把i-size那个元素从堆中移除，然后将当前元素加入以后，再次选择堆定元素加入res。代码简洁很多。

代码 t85 s9 java

注意可以使用lambda表达式更容易地实现上面comparator。

PriorityQueue heap = new PriorityQueue<>((o1, o2) -> o2 - o1); /* 大顶堆 */
