153. 寻找旋转排序数组中的最小值

发表于 2021-04-08 | 分类于 leetcode

难度 medium
已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次 旋转 后，得到输入数组。例如，原数组 nums = [0,1,2,4,5,6,7] 在变化后可能得到：
若旋转 4 次，则可以得到 [4,5,6,7,0,1,2]
若旋转 4 次，则可以得到 [0,1,2,4,5,6,7]
注意，数组 [a[0], a[1], a[2], …, a[n-1]] 旋转一次 的结果为数组 [a[n-1], a[0], a[1], a[2], …, a[n-2]] 。

给你一个元素值 互不相同 的数组 nums ，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的 最小元素 。

示例 1：

输入：nums = [3,4,5,1,2]
输出：1
解释：原数组为 [1,2,3,4,5] ，旋转 3 次得到输入数组。
示例 2：

输入：nums = [4,5,6,7,0,1,2]
输出：0
解释：原数组为 [0,1,2,4,5,6,7] ，旋转 4 次得到输入数组。
示例 3：

输入：nums = [11,13,15,17]
输出：11
解释：原数组为 [11,13,15,17] ，旋转 4 次得到输入数组。

提示：

n == nums.length
1 <= n <= 5000
-5000 <= nums[i] <= 5000
nums 中的所有整数 互不相同
nums 原来是一个升序排序的数组，并进行了 1 至 n 次旋转

解题思路：这道题可以用二分查找的思路解决问题，left初始化为0，right初始化为nums.length-1，然后用中间的数和最右遍的数比较，这种旋转数组只有两种情况，要么跳变点在左边，右边是有序的，要么跳变点在右边，左边是有序的,跳变点就是最小值所在的点，如果nums[mid]>nums[right]，因为nums[0]和nums[right]是接近的，我们可以判断这里有序的部分出现在左边，因此有left=mid+1;否则即nums[mid]<=nums[right]，此时有序的部分是右边，因此此时有right = mid;直到left<right的条件不满足，我们返回nums[right]，为什么是nums[right]，因为最后一个循环必然是left=mid=right-1,如果nums[mid]>nums[right]，则根据上半部分，有left=mid+1,此时退出，right所在位置为最小值，而如果是nums[mid]<=nums[right]，根据前面的写法，此时有right=mid，并且退出，因此right所在位置肯定是最小值。
