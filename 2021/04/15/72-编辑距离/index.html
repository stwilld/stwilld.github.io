72. 编辑距离

发表于 2021-04-15 | 分类于 leetcode

难度 hard
给你两个单词 word1 和 word2，请你计算出将 word1 转换成 word2 所使用的最少操作数 。

你可以对一个单词进行如下三种操作：

插入一个字符
删除一个字符
替换一个字符

示例 1：

输入：word1 = “horse”, word2 = “ros”
输出：3
解释：
horse -> rorse (将 ‘h’ 替换为 ‘r’)
rorse -> rose (删除 ‘r’)
rose -> ros (删除 ‘e’)
示例 2：

输入：word1 = “intention”, word2 = “execution”
输出：5
解释：
intention -> inention (删除 ‘t’)
inention -> enention (将 ‘i’ 替换为 ‘e’)
enention -> exention (将 ‘n’ 替换为 ‘x’)
exention -> exection (将 ‘n’ 替换为 ‘c’)
exection -> execution (插入 ‘u’)

提示：

0 <= word1.length, word2.length <= 500
word1 和 word2 由小写英文字母组成

解题思路：这道题可以用动态规划的思路来解答，看了grandyang的题解，看了好一会才懂的。word1的长度为m，word2的长度为n，dp数组的大小为(m+1)*(n+1)，dp[i][j]表示的是word1前i个字符转换成word2的前j个字符所需要的最少的转换次数。我们分类讨论一下（注意数组索引和字符顺序的差别），如果word1[i-1]=word2[j-1]，则有dp[i][j] = dp[i-1][j-1]，因为此时不需要增加任何操作，而如果word1[i]!=word2[j]，此时有三种情况：1.在word1中增加一个word2[j]，然后直接比较word1[i]和word2[j+1];2.将word1[i]删除，直接跳过，比较word[i+1]和word2[j]；3.将word1[i]替换成word2[j]，直接跳过，比较word[i+1]和word2[j+1]。以字符串”bbc”和“abcd”为例，其dp数组如下所示：

对于dp数组第一行和第一列的数据，我们需要提前确定，这个很容易，因为此时有一个字符串为空，因此从这个字符得到另一个字符串的操作次数就是后者的长度，因此我们就用索引来填充数组即可。
如果word1[i]和word2[j]相等，有dp[i][j] = dp[i-1][j-1]，已经可以很明显地观察到，此时不再讨论，主要分析二者不等的情况，先说一下结论：左边对应插入情况，上边对应删除情况，左上对应替换情况，以第3行第5列的3为例，此时比较的是”bb”和”abcd”，

  • 左边是”bb”转换到”abc”

    此时相当于知道bb->abc的代价为m，求bb到abcd的代价为多少，为了将bb转换为abcd，我们可以在bb后边加上一个d，需要一个插入操作，此时bb->abc的代价已知，因此可以求出bb转换到abcd的代价，此时对应的就是插入情形；

  • 上边是”b”和”abcd”

    此时问题相当于如果知道b->abcd的代价为m，求bb到abcd的代价为多少，为了将bb转换为abcd，我们可以将bb中的第二个b删除掉，需要一个删除操作，此时剩下一个b，转换到abcd的代价已知，因此可以求出bb转换到abcd的代价，此时对应的就是删除情形；

  • 左上是”b”转换到”abc”

    此时相当于知道b->abc的代价为m，求bb到abcd的代价为多少，为了将bb转换为abcd，我们可以将bb的第二个字符替换成d，需要一个替换操作，此时b->abc的代价一直，因此可以求出bb转换到abcd的代价，此时对应的就是替换情形；

因此当word1[i]!=word2[j]的时候，有dp[i][j]=min(dp[i-1][j-1], min(dp[i][j-1], dp[i-1][j]))。我们分类讨论，最后给出dp[m][n]即可。
