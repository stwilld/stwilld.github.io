1090. 受标签影响的最大值

发表于 2021-05-11 | 分类于 leetcode

难度：medium
我们有一个项的集合，其中第 i 项的值为 values[i]，标签为 labels[i]。

我们从这些项中选出一个子集 S，这样一来：

|S| <= num_wanted
对于任意的标签 L，子集 S 中标签为 L 的项的数目总满足 <= use_limit。
返回子集 S 的最大可能的 和。

示例 1：

输入：values = [5,4,3,2,1], labels = [1,1,2,2,3], num_wanted = 3, use_limit = 1
输出：9
解释：选出的子集是第一项，第三项和第五项。
示例 2：

输入：values = [5,4,3,2,1], labels = [1,3,3,3,2], num_wanted = 3, use_limit = 2
输出：12
解释：选出的子集是第一项，第二项和第三项。
示例 3：

输入：values = [9,8,8,7,6], labels = [0,0,0,1,1], num_wanted = 3, use_limit = 1
输出：16
解释：选出的子集是第一项和第四项。
示例 4：

输入：values = [9,8,8,7,6], labels = [0,0,0,1,1], num_wanted = 3, use_limit = 2
输出：24
解释：选出的子集是第一项，第二项和第四项。

提示：

1 <= values.length == labels.length <= 20000
0 <= values[i], labels[i] <= 20000
1 <= num_wanted, use_limit <= values.length

解题思路：这道题目想了大概有20分钟，最终写完通过大概总共耗时35~40分钟吧，还是太慢了。说一下解题思路，值和标签之间具有对应关系，这个关系本来想用HashMap来表示，但是后来想想values中可能有重复的元素会对应不同的标签，因此HashMap中就无法记录两个映射关系，因此放弃了。还有一个注意点就是我们基本还是使用贪心的策略，把一群最大的value加到结果中，同时保证总个数在num_limited里面，每个标签使用次数在use_limit里面，因此我们最好有一个排序的数据结构，让我们能一直取到最大的元素，然后累加标签进行判断即可，这里用的是优先队列，存储的是values到labels的一一对应的数组，然后以第一个元素的大小从大到小进行排序，每次取堆顶元素，用一个map存储标签和使用次数的映射，然后判断其标签是否在map中，如果是，判断其使用次数是否已经达到最大次数use_limit，如果是，就跳过，如果不是，就把相应的值加一，把value加到结果res中，如果标签不在map中，就把值value加到res中，然后把对应的映射关系添加到map中，键为标签，值为1，最后，当我们已经在结果中加了num_limit个数或者优先队列已经为空的时候，就退出循环。返回结果即可。
