686-重复叠加字符串匹配

发表于 2021-12-22 | 分类于 leetcode

难度 medium

链接：https://leetcode-cn.com/problems/repeated-string-match

题目描述：

给定两个字符串 a 和 b，寻找重复叠加字符串 a 的最小次数，使得字符串 b 成为叠加后的字符串 a 的子串，如果不存在则返回 -1。

注意：字符串 “abc” 重复叠加 0 次是 “”，重复叠加 1 次是 “abc”，重复叠加 2 次是 “abcabc”。

示例 1：

输入：a = “abcd”, b = “cdabcdab”
输出：3
解释：a 重复叠加三遍后为 “abcdabcdabcd”, 此时 b 是其子串。
示例 2：

输入：a = “a”, b = “aa”
输出：2
示例 3：

输入：a = “a”, b = “a”
输出：1
示例 4：

输入：a = “abc”, b = “wxyz”
输出：-1

提示：

1 <= a.length <= 104
1 <= b.length <= 104
a 和 b 由小写英文字母组成

解题思路：

这道题目不是很难，只是case太多，有一些很难想到。为了使字符串b成为叠加后的字符串a的子串，首先a至少要叠加到长度大于等于b，这个是叠加的最少次数，记为c1，但还有一些特殊情况，比如示例1，a叠加两次后变为”abcdabcd”，和字符串b即”cdabcdab”长度相等，但此时第一个匹配的字符并不是a的起始字符，因此至少要再叠加一次a，此时记叠加次数为c2，c2=c1+1，这是最多的叠加次数了。因此我们将字符串a直接叠加c2次，然后寻找字符串b在a中的初始匹配位置idx，如果idx+字符串b的长度大于字符串a的长度乘以c1，那说明前c1个a字符串的叠加结果无法包含字符串b，因此需要c1+1个（如a=”abcd”，b=”cdabcdab”，此时返回3），否则c1个a字符串的叠加已经包含字符串，直接返回c1即可（如a=”abcd”，b=”bcdab”，此时返回2即可）。

代码： java t51 s67
