786-第 K 个最小的素数分数

发表于 2021-12-13 | 分类于 leetcode

难度 hard

链接：https://leetcode-cn.com/problems/k-th-smallest-prime-fraction/

题目描述：

给你一个按递增顺序排序的数组 arr 和一个整数 k 。数组 arr 由 1 和若干 素数 组成，且其中所有整数互不相同。

对于每对满足 0 < i < j < arr.length 的 i 和 j ，可以得到分数 arr[i] / arr[j] 。

那么第 k 个最小的分数是多少呢? 以长度为 2 的整数数组返回你的答案, 这里 answer[0] == arr[i] 且 answer[1] == arr[j] 。

示例 1：

输入：arr = [1,2,3,5], k = 3
输出：[2,5]
解释：已构造好的分数,排序后如下所示:
1/5, 1/3, 2/5, 1/2, 3/5, 2/3
很明显第三个最小的分数是 2/5
示例 2：

输入：arr = [1,7], k = 1
输出：[1,7]

提示：

2 <= arr.length <= 1000
1 <= arr[i] <= 3 * 104
arr[0] == 1
arr[i] 是一个 素数 ，i > 0
arr 中的所有数字 互不相同 ，且按 严格递增 排序
1 <= k <= arr.length * (arr.length - 1) / 2

解题思路：

这道题目总体难度还是不大的，就是需要熟练运用数据结构，这里用了固定大小的大顶堆PriorityQueue<int[]>，我们自定义其比较函数，使其能够对添加到堆中的数组int[]进行排序，这里的数组就是一个分数的数组表示，int[0]表示分子，int[1]表示分母，我们使最大的一个位于堆顶。在定义好大顶堆之后，开始用嵌套循环遍历数组arr，组合成一个分数，用int数组表示，如果堆的size小于k，直接添加到堆中，否则将我们当前组合成的分数与堆顶元素进行比较，如果当前分数小于堆顶元素，则将堆顶元素弹出，并把我们当前组合的分数添加到堆中，随后堆对其进行排序，从而使堆中最大的分数仍处于堆顶。当遍历完所有组合的分数后，堆顶元素就是最k小的素数分数了。这个解法本质上也是暴力解。

代码： java t100 s100
