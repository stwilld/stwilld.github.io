689-三个无重叠子数组的最大和

发表于 2021-12-13 | 分类于 leetcode

难度 hard

链接：https://leetcode-cn.com/problems/maximum-sum-of-3-non-overlapping-subarrays

题目描述：

给你一个整数数组 nums 和一个整数 k ，找出三个长度为 k 、互不重叠、且 3 * k 项的和最大的子数组，并返回这三个子数组。

以下标的数组形式返回结果，数组中的每一项分别指示每个子数组的起始位置（下标从 0 开始）。如果有多个结果，返回字典序最小的一个。

 

示例 1：

输入：nums = [1,2,1,2,6,7,5,1], k = 2
输出：[0,3,5]
解释：子数组 [1, 2], [2, 6], [7, 5] 对应的起始下标为 [0, 3, 5]。
也可以取 [2, 1], 但是结果 [1, 3, 5] 在字典序上更大。
示例 2：

输入：nums = [1,2,1,2,1,2,1,2,1], k = 2
输出：[0,2,4]

提示：

1 <= nums.length <= 2 * 104
1 <= nums[i] < 216
1 <= k <= floor(nums.length / 3)

解法一
解题思路：
这道题目可以使用滑动窗口的解法，使用3个大小为k的窗口，sum1表示第一个滑动窗口的元素和，该滑动窗口从[0, k-1]开始，sum2为第二个滑动窗口的元素和，该滑动窗口从[k, 2k-1]开始，sum3为第三个滑动窗口的元素和，该滑动窗口从[2k, 3k-1]开始，我们同时向右移动三个窗口，维护sum1的最大值maxSum1以及对应的初始位置maxIdx1，当滑动时，计算当前maxSum1与sum2之和，用maxSum12维护其最大值，并更新此时第一个窗口和第二个窗口的起始位置maxSum12Idx1, maxSum12Idx2；再计算maxSum12和sum3之和，用totalSum维护其最大值，同时更新三个窗口的起始位置，最后返回结果。
对于题目要求的最小字典序，由于我们是从左向右遍历的，并且仅当元素和超过最大元素和时才修改最大元素和，从而保证求出来的下标列表是字典序最小的。

代码： java t60 s75

maxSum12Idx1 = maxIdx1的意义在于：防止窗口1和窗口2重合，以nums = [4,5,10,6,11,17,4,11,1,3], k = 1 输出：[4,5,7]为例，当第一个窗口包含11，第二个窗口包含17，第三个窗口包含4时，有maxSum1 = 11, maxSum12 = 28, totalSum = 32，此时继续滑动窗口，maxSum1 = 17, 此时sum2 = 4， sum2 + maxSum1 = 21 < maxSum12 = 28，所以此时不会运行中括号里的内容：

也就是尽管maxSum1更大了，也更新了maxIdx1，但此时maxSum12Idx1并不会更新，还是保持在原来的位置，这就保证了maxSum12是在维持前两个窗口之和最大时记录下来的两个窗口的起始值，保证两个窗口不会有重合的地方，在最后给res赋值的时候，也是用res[0] = maxSum12Idx1这样的语句，而不是res[0] = maxIdx1这样的语句，假如没有maxSum12Idx1 = maxIdx1这一句，且最后用res[0] = maxIdx1这个赋值语句，则输出结果是[5, 5, 7]。

解法二
解题思路：
此题也可以使用动态规划的思路进行求解，我们用dp[i][j]表示到第i个元素位置，前j个互不重叠的子数组的最大值，对于当前的第i个元素对应的值，我们有取和不取两种选择，如果不取该元素，则当前dp[i][j]值为到i-1个元素时前j个互不重叠的子数组的最大值，即dp[i-1][j]，如果选取该元素，则值为到i-k个元素时前j-1个互不重叠的子数组的最大值dp[i-k][j-1]加上最后一段包含第i个元素的子数组的和，我们在两者中选择较大者即可，可以得到状态转移方程如下:
dp[i][j] = max(dp[i-1][j], dp[i-k][j-1] + sum[i] - sum[i-k])
其中sum为前缀和数组，表示num[0] + … + num[i-1]

代码：java t60 s75
