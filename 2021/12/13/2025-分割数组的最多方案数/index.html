2025-分割数组的最多方案数

发表于 2021-12-13 | 分类于 leetcode

难度 hard

链接：https://leetcode-cn.com/problems/maximum-number-of-ways-to-partition-an-array

题目描述：

给你一个下标从 0 开始且长度为 n 的整数数组 nums 。分割 数组 nums 的方案数定义为符合以下两个条件的 pivot 数目：

1 <= pivot < n
nums[0] + nums[1] + … + nums[pivot - 1] == nums[pivot] + nums[pivot + 1] + … + nums[n - 1]
同时给你一个整数 k 。你可以将 nums 中 一个 元素变为 k 或 不改变 数组。

请你返回在 至多 改变一个元素的前提下，最多 有多少种方法 分割 nums 使得上述两个条件都满足。

示例 1：

输入：nums = [2,-1,2], k = 3
输出：1
解释：一个最优的方案是将 nums[0] 改为 k 。数组变为 [3,-1,2] 。
有一种方法分割数组：

  • pivot = 2 ，我们有分割 [3,-1 | 2]：3 + -1 == 2 。
    示例 2：

输入：nums = [0,0,0], k = 1
输出：2
解释：一个最优的方案是不改动数组。
有两种方法分割数组：

  • pivot = 1 ，我们有分割 [0 | 0,0]：0 == 0 + 0 。
  • pivot = 2 ，我们有分割 [0,0 | 0]: 0 + 0 == 0 。
    示例 3：

输入：nums = [22,4,-25,-20,-15,15,-16,7,19,-10,0,-13,-14], k = -33
输出：4
解释：一个最优的方案是将 nums[2] 改为 k 。数组变为 [22,4,-33,-20,-15,15,-16,7,19,-10,0,-13,-14] 。
有四种方法分割数组。

提示：

n == nums.length
2 <= n <= 105
-10^5 <= k, nums[i] <= 10^5

解题思路：

这道题目还是很有难度的，题解非常巧妙。对于数组nums，记录其前缀和数组sum，记整个数组的和为total，分情况讨论：

  • 如果不改变某个元素的值，则其需要满足$sum[i]=total-sum[i]$，也就是$sum[i] = \frac{total}{2}$，也就是我们需要记录前缀和数组中，和total/2相等的元素的个数，可以用HashMap记录。
  • 如果改变某个元素的值，假设当前遍历到第i个元素，则改变的值为d = k - nums[i]，
    • 对于j<i，其前缀和数组并不受影响，此时为了满足分割条件，要求$sum[j] = total + d - sum[j]$，即$sum[j] = \frac{total+d}{2}$​；
    • 对于j>=i，其前缀和数组都增加了d，此时为了满足分割条件，要求$sum[j]+d = total+d-(sum[j]+d)$，即$sum[j] = \frac{total-d}{2}$;

因此，这一道题的解法就是，用两个HashMap记录前缀和sum[i]和出现次数的关系，一个mapR记录当前元素右边的映射关系，一个mapL记录当前元素左边的关系，由于我们从左到右遍历数组nums，因此mapR一开始记录了所有前缀和-次数的映射关系，在计算sum前缀和数组的时候，我们就可以往mapR中添加这些映射关系，遍历后开始讨论第一种情况，此时分割满足$sum[i] = \frac{total}{2}$，因此我们将结果res初始化为mapR.getOrDefault(total/2, 0)，随后开始讨论修改元素的情况，如果修改了当前元素，新增加的情况为：mapL.getOrDefault((total+d)/2, 0) + mapR.getOrDefault((total-d)/2, 0)，我们用这个值和res中的较大值更新res即可。需要注意，在用map取值的时候，key为整数，因此需要在total为偶数以及(total+d)为偶数的情况下，才可以从map中取值，此外，在更新了当前元素之后，我们向后移动，此时mapL中sum[i]对应的值要加一，mapR中sum[i]对应的值要减一，动态更新HashMap的变化。
