825-适龄的朋友

发表于 2021-12-27 | 分类于 leetcode

难度 medium

链接：https://leetcode-cn.com/problems/friends-of-appropriate-ages

题目描述：

在社交媒体网站上有 n 个用户。给你一个整数数组 ages ，其中 ages[i] 是第 i 个用户的年龄。

如果下述任意一个条件为真，那么用户 x 将不会向用户 y（x != y）发送好友请求：

age[y] <= 0.5 * age[x] + 7
age[y] > age[x]
age[y] > 100 && age[x] < 100
否则，x 将会向 y 发送一条好友请求。

注意，如果 x 向 y 发送一条好友请求，y 不必也向 x 发送一条好友请求。另外，用户不会向自己发送好友请求。

返回在该社交媒体网站上产生的好友请求总数。

示例 1：

输入：ages = [16,16]
输出：2
解释：2 人互发好友请求。
示例 2：

输入：ages = [16,17,18]
输出：2
解释：产生的好友请求为 17 -> 16 ，18 -> 17 。
示例 3：

输入：ages = [20,30,100,110,120]
输出：3
解释：产生的好友请求为 110 -> 100 ，120 -> 110 ，120 -> 100 。

提示：

n == ages.length
1 <= n <= 2 * 104
1 <= ages[i] <= 120

解题思路1：

这道题目说实话有点费解，以下三个条件中，条件3是隐含在条件2中的：

  1. age[y] <= 0.5 * age[x] + 7
  2. age[y] > age[x]
  3. age[y] > 100 && age[x] < 100

由于题目中说的是，只要有一个条件为真，即不会发送，因此也即只有所有条件都为假，x才会向y发送信息，此时即需要满足以下条件：

  1. 0.5*age[x]+7 < age[y]
  2. age[x]>=age[y]

于是有0.5*age[x]+7 < age[y]<=age[x]，得出age[x]>14，所以只需要考虑大于14的数，首先对数组进行排序，遍历大于14的所有数，记当前遍历到的元素为age，然后用双指针维护一个区间，当ages[left]<=age*0.5+7，则向右移动left指针，直到ages[left]>age*0.5+7，若ages[right+1]<=age，则向右移动right指针，直到ages[right+1]>age，此时闭区间[left, right]内的所有数都满足条件，其长度为(right - left + 1)，但需要剔除age自身，因此在结果中加上(right-left)即可。

代码： java t29 s14

解题思路2：

下面这个解法是我自己写的，比较麻烦，主要是思路也是先对ages数组进行排序，遍历ages数组，记当前遍历到的元素的下标为i，然后往前找到和当前元素相等的最靠前元素的下标j，从j到i这一段，是双方要互发的，因为都是相等的元素；然后用二分查找法，找到最后一个满足ages[tmp]<=ages[i]*0.5+7的元素的下标，从tmp到j-1这一段，是只发一次的，也就是从i发出，tmp到j-1之间的元素接收。在结果中把这两种情况加进去即可。此外，还要注意小于15的元素是不考虑的，因此一开始也可以用二分查找法找出小于等于15的最后一个元素，然后只保留后面的数组部分。

还是看官方题解吧，这个思路我尝试了快两个小时，只能说当作顺便复习了二分查找吧。

代码： java t6 s58
